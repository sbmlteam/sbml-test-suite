How do I run a test?

When the Test Suite is loaded go to the File menu and select "New Test".

A wizard will start where you can set the configurations for the test you'd like to run by selecting level, test type, component tags and test tags to exclude and the path/command for your wrapper.

Turn logging on or off via the checkbox in the preferences submenu item "Generate debugging log". Severe messages are always logged while warning and info level messages are only logged if the "Generate debugging log" item is selected.

Then select "Finish". The wizard will close and the green start button in the main test window will be active - select this button to begin your test. You can choose to view the results via the list view or the map view.

Where are my files?

The first time the stand alone SBML Test Suite is run it creates a hidden directory in the user home directory called .sbmltestrunner. During the first run the SBML test cases are unzipped into userhome/.sbmltestrunner/sbml-test-suite/cases.

The output from your wrapper can be found in userhome/.sbmltestrunner/wrapperOutput with filenames in the format of resultsnnnnn.csv where nnnnn is the specific test case that was run through your wrapper.

The log file can be found at userhome/.sbmltestrunner/sbml-test-suite.log. Only severe messages are logged unless the debugging log is selected in the preferences menu at which point all messages will be printed to the log file.

Test Cases

Each test case directory is named according to the test case it represents following a naming scheme of 5 digits. Each directory contains:

XXXXX-model.m - which contains the tags, the description and the MathSBML code used to create the model.

XXXXX-plot.jpg and XXXXX-plot.eps - graphs of the expected output

XXXXX-results.csv - the data used in the testing

XXXXX-sbml-l1v2.xml
XXXXX-sbml-l2v1.xml
XXXXX-sbml-l2v2.xml
XXXXX-sbml-l2v3.xml
The SBML files; directories will only contain relevant levels/versions so will always have l2v2 and l2v3 but will only have l1v2 and l2v1 where appropriate.

XXXXX-settings.txt The settings file. These all inlcude:
start:0
duration:5.0
steps:50
variables: S1,S2
absolute: 0.000000000000001
relative: 0.0001
These variables map to the start time of simulation, duration and number of points (steps) expected in the output, the variables that should ouptput in the order in which we expect them to occur, the tolerances that we will use to determine pass/fail. The absolute tolerance is 0.0001 times the maximum initial variable value within the model. (In the case above the initial amount used was 1e-15) The relative tolerance is always 0.0001.

How do I update the test cases

Update test cases by the test suite menu by selecting "Update test cases". The program will then connect to the interenet and check sourceforge to see if there is a newer version of the test cases available than the one that shipped with your application. If there is it will automatically update your copy - if not it will notify you that you have the most current version available.

About the SBML Test Suite Runner

The SBML Test Suite allows anyone to evaluate the degree of SBML support implemented in an SBML-compatible software package. This stand alone service allows you to evalute your command line invokable software program with SBML test cases through a self written wrapper.

The stand alone SBML Test Suite Runner is an interface to the full SBML Test Suite, written by Sarah M. Keating, Kimberly Begley and Michael Hucka.
