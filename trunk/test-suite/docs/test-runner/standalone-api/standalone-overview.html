This is the Programmer's Manual for the SBML Test Suite Standalone application; it describes the internal aspects of the application for programmers interested in reusing or extending the codebase.

Authors: The SBML Test Suite Standalone Application and its documentation was written by Kimberly Begley.

The complete SBML Test Suite was written by Sarah Keating, Kimberly Begley and Michael Hucka. It is based in part on the SBML Semantic Test Suite written by Andrew Finney.

General information

The SBML Test Suite is a conformance testing system for SBML. It allows developers and users to test the degree and correctness of SBML support provided in an SBML-compatible software package. The Test Suite provides a large corpus of valid SBML models, each with representative, simulated time course data, plot outputs, simulation settings, documentation, and other information. Each test case exercises different combinations of SBML features and components.

The Standalone Application component of the SBML Test Suite is a test automation system. It allows a developer or end user to run a simulation program against the SBML Test Suite cases, collect the results, compare them to expected results, and report on the successes and failures.

Communication, bug reports, and related topics

Please report bugs and other issues with all aspects of the SBML Test Suite using the tracker at http://sbml.org/software/test-suite/issue-tracker.

You can also report problems on the sbml-interoperability mailing list. This may even have advantages, such as that other people may also have experienced the issue and offer a workaround more quickly than the SBML Test Suite developers can respond.

For more information about SBML, please visit http://sbml.org/ on the Internet.
